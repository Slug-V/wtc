# 勤務配分（wtc）

> worktime calculator for a 6h/8h worker which normalizes monthly salary

「8時間勤務してもよい6時間契約」という働き方で、**月給を一定に保つ**ことを支援する勤務時間管理アプリです。各日を **6h / 8h / 残業 / 欠勤** のどれにすべきかを示し、月末の手取りが一定の月給に着地するよう導きます。依存ライブラリ・ビルド・サーバーは一切なく、**単一の `index.html` だけ**で動作する PWA です。

## これは何か

- 基準は「**6時間契約・基準日数18日**」。基準18日をすべて8hで埋めると、ちょうど月給（時給 × 8h × 18日）に一致します。これが整合性の検算点です。
- 月ごとに「6hにすべき日数 / 8hにすべき日数」が決まっており、その配分を崩さなければ月給は一定に保たれます。アプリは記録を集計し、「あと何日を6h（または8h）にすればよいか」を一目で示します。
- スマホのホーム画面に追加してスタンドアロン動作することを想定したダークテーマ UI。データはすべて端末の `localStorage` に保存されます。

## 主な機能

- **その日の記録**：ワンタップで `6h`（6時間勤務／有休）・`8h`・残業（`+1h` / `+2h` / 分指定の端数）・`欠勤` を登録。
- **月次ダッシュボード**：その月のモードに応じて表示が変化。
  - `6h残枠` … あと何日を6hにできるか（残り日数）
  - `8hノルマ` … あと何日を8hにすべきか（残り日数）
  - `残業ノルマ`（基準割れ月）… 必要残業時間に対する達成ゲージ（割増1.25倍）
- **今日の発生額（リアルタイム）**：時給と勤務時間帯を設定しておくと、今日の勤務時間に沿って稼いだ額が時々刻々と増えていく様子を表示。休憩中は加算が止まり、残業帯は割増で速く増えます。
- **今月の確定額**：勤務を完走した日の合計を表示。今日分はその日の勤務を終えた時点で算入されます。
- **記録の訂正**：直近14日分を一覧し、種別を選び直して修正。
- **設定・バックアップ**：時給／勤務時間帯の設定、記録件数、JSON での書き出し・読み込み、全データ消去。

## 仕組み（ドメインモデル）

- **6h残枠モードと8hノルマモードは同一状態の別表示**です。別の雇用状況ではありません。月内で6h化する日数が総勤務日数の半分以下なら「6h残枠」、半分超なら「8hノルマ」として、見せる数が小さくなる側を表示します。この判定は年度データの時点で**各月固定**であり、月の途中で動的に切り替わることはありません（意図的な設計）。
- **基準割れ月（例：8月）は残業ノルマモード**。残業でしか月給を埋められないため、要残業時間を「不足時間 ÷ 1.25（割増）」として算出し、ゲージで進捗を示します。
- **有休は6h相当で給与確定**します（フルタイム8h扱いではない）。そのため有休も6hボタンで記録され、金額も6h分で正しくなります。
- **欠勤は無給**。さらに月給維持の観点から補正がかかります（6h残枠月では残枠が計4減る／8hノルマ月ではノルマが3増える／残業ノルマ月では要残業に8h分が上乗せ）。

## 設定とデータ

- **時給**：未入力なら金額系 UI は一切表示されません。
- **勤務時間帯**：始業・休憩開始・休憩終了。8h以上の日のみ休憩を無給として差し引き、6hの日は休憩なしで連続計算します。未設定ならリアルタイム表示は行いません。
- 時給・勤務時間帯ともに**この端末の `localStorage` にのみ保存**され、バックアップ JSON にも含まれます（読み込みで復元）。

## プライバシー

公開リポジトリのため、**金額（時給・月給）や勤務時刻をソースコードに一切ハードコードしません**。これらはユーザーが入力し、端末ローカルにのみ保存されます。

## 技術

- 単一ファイル（HTML / CSS / JavaScript をすべて `index.html` に内包）、バニラ JavaScript、外部依存なし。
- PWA（data URI による manifest / apple-touch-icon、セーフエリア対応、`standalone` 表示）。
- `localStorage` キー：記録 `worktime.records.v1` ／ 設定 `worktime.settings.v1`。
- 年度別の勤務日数データ（2026-04〜2027-03）を `index.html` 内の `YEAR_DATA` にハードコード。

## 使い方・デプロイ

静的な単一ファイルなので、`index.html` をブラウザで開くだけで動きます。GitHub Pages で配信する場合はサブパス（例：`https://<user>.github.io/wtc/`）になる前提で作られているため、リソース参照に先頭スラッシュの絶対パスは使いません。

## 制限事項

- データは端末ローカルのみ。クラウド同期はなく、バックアップは手動の JSON 書き出し／読み込みに依ります。
- 月別の勤務日数・配分はコードに直書きのため、年度更新にはコード修正が必要です。
